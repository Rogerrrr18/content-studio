# CLAUDE.md — content-studio 媒体运营工作台

## 项目用途
个人媒体矩阵（1 公众号 + 3 小红书账号）的选题、生产、分发、复盘、私域转化全流程工作台。

## 矩阵账号

| 账号 | 平台 | 定位 | 转化目标 |
|------|------|------|----------|
| 主号 | 公众号 | 海外科技深度 + 热点 + benchmark | 私域引流 |
| 个人 IP 号 | 小红书 | 真人 + 教程 + 观点（视频为主）| 商单 / 个人影响力 |
| 知识/资讯号 | 小红书 | 海外播客 + 公众号长文转图文（纯客观）| 涨粉 / 矩阵引流 |
| 产品评测号 | 小红书 | PM + benchmark + 软文 | 软文变现 |

完整 SOP 见 `WORKFLOW.md`，本周选题见 `01_选题库/_近期推荐_*.md`。

## 工作时的默认行为

1. **写公众号文章**：用 `wechat-article-writer` skill。排版 / 发草稿用 `wechat-studio`。
2. **小红书封面**：用 `atutun-xhs-cover` skill。
3. **新选题**：从 `templates/选题卡.md` 复制；存到 `01_选题库/<账号>/YYYYMMDD_标题.md`。
4. **素材入库**：每条必须带原始链接 + 日期。
5. **写跨账号联动**：默认按"公众号长文 → 知识号图文 → IP 号视频 → 评测号软文"四联模式提出方案。
6. **海外内容口径**：聚焦海外科技公司动态（OpenAI/Anthropic/Google）、海外长播客（Lex/Dwarkesh/Latent Space）、海外 indie 案例。

## 不要做的事

- 不要把 `outputs/`、视频、原图加进 git（已在 `.gitignore`）。
- 不要把私域客户名单、合同、商务合作明细写进会被推到 GitHub 的文件。
- 软文标记只在内部 frontmatter 里写，不出现在对外文案。
- 写文案别堆 emoji（除非用户明确要）。
